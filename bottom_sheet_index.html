1. Bottom Sheet Menu 互動規範

本區塊說明在 Device Manager 中，於 Bottom Sheet 內使用 List Menu 時的核心規範。由於手機端空間有限且通常具備過濾、設定等目的，控制元件、背景狀態與執行時機的定義必須非常精確，以免造成 IT 管理員在維護大量設備時產生操作上的困惑。

3. 💡 UX 專家重點解析

🚩

複合元件熱區切分 (Hybrid Row)

當 List 本身可被選也可做為入口時，只需定義並切分點擊範圍。左側控制項區域負責「選取」，右側文字/箭頭負責「導航」。絕不能點擊整列同時觸發兩種行為。

⚠

狀態保留 (非必要，視情況)

並非所有帶控制項的選單都必須永久保留狀態。若為「全域過濾器」應保留；若為「一次性派發任務」，則下次展開時應重設狀態，避免歷史選項干擾。

✔

雙重 Active 避讓原則

只要有單選/複選控制項，就不需要給予整列背景色（避免雙重反饋），僅需保留按下瞬間的 Ripple 水波紋。純導航入口則可透過背景色標示 Focus 狀態。

4. Menu 元件設計判斷流程

此流程圖定義了在設計 Bottom Sheet 內的清單項目時，何時該給予控制項、何時該保留狀態，以及如何處理複合型入口。

🔍 提示：您可以使用滑鼠滾輪縮放，或按住左鍵拖曳移動視角。

graph TD
    Start([開始: 設計 Bottom Sheet Menu]) --> Q1{"是否具備「選擇」行為？"}

    %% ================= 路徑：需要控制項 =================
    Q1 -- "是" --> Q_SelectionMode{"選擇情境與模式為何？"}

    Q_SelectionMode -- "明確的單選" --> Radio["使用 Radio Button (單選)"]
    Q_SelectionMode -- "明確的複選" --> Checkbox["使用 Checkbox (複選)"]
    Q_SelectionMode -- "輕量的單/複選" --> CheckIcon["使用 Check Icon (輕量標示)"]

    Radio --> Q_IsEntry{"是否同時作為
「階層選單的入口」？"}
    Checkbox --> Q_IsEntry
    CheckIcon --> Q_IsEntry

    Q_IsEntry -- "是 (複合型)" --> Hybrid["複合型元件 (Hybrid Row)
左放控制項，右放箭頭 (＞)
[注意] 明確切分點擊熱區"]
    Q_IsEntry -- "否 (純選擇)" --> PureSelect["純選擇元件
點擊整列皆為切換選取狀態"]

    PureSelect --> Q_Persistence{"是否需保留狀態？"}
    Hybrid --> Q_Persistence

    Q_Persistence -- "視情況保留" --> StateKeep["若為全域設定(如過濾)：保留狀態
若為一次性任務(如指定目標)：不保留"]

    %% ================= 路徑：不需要控制項 =================
    Q1 -- "否" --> NoControl["不需要控制項 (No Control btn)"]
    NoControl --> Q_Action{"該元件的互動目的為何？"}

    Q_Action -- "階層選單的入口" --> NavEntry["純文字 + 箭頭 (＞)
點擊整列引導進入下一層"]
    Q_Action -- "執行某一動作" --> ActionExec["純文字 / 動作圖示
不具備選取狀態，點擊即執行"]
